Visão Geral

O Logs Integração SAJ é uma aplicação web local para consulta, análise e exportação dos XMLs de integração judicial gerados pelo logexternalizer SAJ. Ele substitui a necessidade de acessar o servidor diretamente pelo navegador ou pelo SoapUI.

🔷 Logs-HUB

Consulta de mensagens SOAP trocadas pelo HUB de integração. Organizado por PGMP → Operação → Data.

🔶 Logs-MNI

Consulta de XMLs MNI (comunicação direta). Organizado por PGMP → Data.

💾 Consultas Salvas

Repositório de consultas marcadas para referência futura, filtráveis e exportáveis em JSON.

📤 Exportação

Cada arquivo pode ser exportado em XML formatado, PDF ou CSV. Pares completos podem ser exportados em ZIP ou PDF combinado.

Rede SAJ obrigatória: o servidor de logs roda em 172.50.1.164:9999, acessível somente dentro da rede corporativa SAJ ou via VPN. Sem acesso, a aplicação exibirá mensagem de erro explicativa.

Como Iniciar

Opção 1 — arquivo BAT (recomendado)

Dê duplo clique em iniciar.bat na pasta do projeto. O terminal abrirá e o app estará disponível em http://localhost:8501.

Opção 2 — terminal

cd D:\Projetos\pj-consultador-xml
python -m streamlit run app.py
Primeira execução: pode levar alguns segundos para baixar a lista de PGMPs. Se a tela ficar em "Carregando..." por mais de 30 segundos, verifique a conexão com a rede SAJ.

O funcionamento é idêntico ao HUB. A diferença é que não há seleção de Operação — os arquivos de todas as operações ficam juntos na pasta da data.

O campo UUID é visível na tabela, facilitando identificar pares manualmente quando o pareamento automático não encontra correspondência exata.

💾 Consultas Salvas Salvas

Armazena localmente metadados de arquivos importantes para referência futura.

Como salvar uma consulta

  1. Carregue um arquivo (botão 📥 Carregar arquivo + par).
  2. Na seção "Exportar par completo", clique em 💾 Salvar consulta.
  3. A consulta é gravada em saved_queries/ com timestamp automático.

Visualizar consultas salvas

Acesse a seção 💾 Consultas Salvas no menu lateral. Use o campo de filtro para buscar por PGMP, operação ou número de processo.

Exportar todas as consultas

Clique em ⬇️ Exportar todas (JSON) para baixar um arquivo JSON com todas as consultas salvas.

🔍 Busca por conteúdo dos XMLs

Expanda o painel "Buscar dentro do conteúdo dos XMLs". Digite um termo e clique em 🔎 Buscar.

Como funciona: o sistema baixa todos os XMLs da data selecionada e pesquisa o termo em cada um. Os resultados mostram quantas ocorrências foram encontradas e um preview do conteúdo. Os XMLs baixados ficam em cache durante a sessão — buscas subsequentes na mesma data são instantâneas.

Clicando em uma linha do resultado da busca, o arquivo correspondente fica selecionado automaticamente na seção de visualização.

🔍 Busca interna no XML carregado

Após carregar um arquivo, use o campo "🔍 Buscar nos XMLs". Os resultados exibem a tag XML, o valor e o XPath para cada ocorrência encontrada nos dois arquivos (selecionado + par).

Pareamento Automático

Ao selecionar um arquivo, o sistema localiza automaticamente o arquivo oposto (request ↔ response).

Algoritmo de pareamento (em ordem de prioridade)

  • 1
    Mesmo UUID

    Se o nome do arquivo contém um UUID padrão (xxxxxxxx-xxxx-xxxx-xxxx-xxxxxxxxxxxx), o sistema busca o arquivo oposto com o UUID idêntico. É o método mais preciso.

  • 2
    Timestamp mais próximo (fallback)

    Se nenhum UUID for encontrado, o sistema usa o arquivo oposto com o timestamp mais próximo — desde que a diferença seja de até 120 segundos.

O par identificado é exibido em destaque acima da busca interna:

🔗 Response pareado (mesmo UUID): 14-30-05-123_response_abc123.xml
Se nenhum par for encontrado dentro dos critérios acima, a aba do arquivo oposto exibirá "Arquivo não carregado ou não encontrado." Isso pode ocorrer para operações que não possuem response (ex: notificações) ou quando os UUIDs diferem entre request e response.

Data inteira

Clique em 📦 Exportar data inteira (ZIP) para baixar todos os XMLs da data selecionada em um único arquivo ZIP. Útil para análise offline ou arquivamento.

Tabela de arquivos

O botão ⬇️ Exportar tabela (CSV) acima da tabela exporta a listagem de arquivos visível (após filtros) em CSV.

Persistência de Seleções

As últimas seleções são salvas automaticamente em config.json na pasta do projeto. Ao reabrir o app, as seleções anteriores são restauradas.

Campo salvoSeção
PGMP selecionadoHUB e MNI
Operação selecionadaHUB
Data selecionadaHUB e MNI
Último arquivo visualizadoHUB e MNI
Os filtros de busca (nome, tipo, ordenação, busca por conteúdo) não são persistidos — são limpos ao reiniciar. Apenas as seleções de navegação principal (PGMP, Operação, Data, Arquivo) são mantidas.

Erros Comuns

❌ Sem acesso ao servidor HUB / MNI

Causa: máquina fora da rede SAJ ou VPN desconectada.
Solução: conecte-se à VPN SAJ e recarregue a página (F5).

⏳ Timeout ao buscar arquivos

Causa: o servidor pode estar lento ou a data selecionada possui muitos arquivos.
Solução: aguarde o timeout (300s) ou tente uma data com menos volume.

XML exibido em branco ou com erro de parse

Causa: o arquivo pode estar corrompido ou ser do tipo MTOM (multipart).
Solução: o app tenta extrair automaticamente o XML do MTOM. Se a visualização mostrar <!-- Erro -->, o arquivo pode estar vazio no servidor.

PDF não gera / erro de fonte

Causa: as fontes arial.ttf ou consola.ttf não foram encontradas em C:\Windows\Fonts\.
Solução: verifique se as fontes existem no caminho. Em Windows padrão elas sempre estão presentes.

Tabela não sincroniza com o selectbox

Clique diretamente na linha da tabela (não no cabeçalho ou na borda). O Streamlit registra o clique e atualiza o selectbox na recarga automática da página.
